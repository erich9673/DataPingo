Bulk Comments for Jira

Post one comment to up to 25 Jira issues at once, give any ticket a different comment, or copy an existing comment, images, videos and files included, across many tickets.

Overview

Bulk Comments for Jira is an Atlassian Forge app installed directly inside Jira Cloud. It lets you write a rich-text comment and post it to up to 25 issues at once, write a different comment for any individual ticket in the same send, and attach images, videos and files of up to 150 MB each — or copy a comment that already exists, media and all, onto other tickets exactly as-is.

No context switching: Everything happens inside Jira. There's no external server in the data path — comments and media move between Jira tickets through Atlassian's own infrastructure.

Write a comment

The editor is the one you already know from Jira: headings, bold, italics, underline, strikethrough, text colour, bullet, numbered and task lists, quotes, dividers, tables, code blocks with syntax highlighting, emoji, @mentions and slash commands. The comment editor below is the full reference.

By default one comment goes to every selected ticket. Click any ticket in the list to write that issue a different comment — the list shows at a glance which tickets share the comment and which have their own, and only tickets that actually have a comment get posted. So a single send can carry one shared update to twenty tickets and a specific note to five others.

Attach images, videos and files

Drag a file into the editor, paste a screenshot straight from your clipboard, or pick one from the toolbar. Each file can be up to 150 MB — or your site's own Jira attachment limit, whichever is lower — and a comment can carry as many files as you need.

Uploading starts the moment you attach, not when you press Send, so by the time you have finished writing the file is usually already staged and the send is near-instant. Images and video preview inline, can be aligned left, centre or right, and resize by dragging their edge; PDFs and other documents post as file cards. Large files are copied to each ticket in the background, so a long video never stalls the batch.

Copy a comment

Paste the link to an existing comment and copy it exactly as-is onto other tickets. Images, videos and file attachments come along automatically.

Three entry points

Open the full app from the global page in the left sidebar, or start right from an issue: the Bulk Comments option in an issue's ••• actions menu opens the comment editor already seeded with that ticket, and the Bulk Comments tab in the issue's Activity panel auto-selects the most recent comment to copy to other tickets.

Full Guide

Writing a comment

Writing a comment walks through three steps — Tickets, Comment, then Send. Pick your issues first (up to 25), then write with rich-text formatting — bold, italics, lists, and @mentions. Each comment is posted individually to its ticket.

On the Write step, choose how you want to comment:

  • One comment for all — write it once and post the identical text to every selected ticket.
  • A different comment — click any ticket in the list to write it something else, without leaving the page. The list shows which tickets share the comment and which have their own.

The comment editor

Everything you write is posted as native Jira formatting, so a comment sent from here is indistinguishable from one typed into Jira itself — same headings, same tables, same code blocks, same file cards. The toolbar and the keyboard shortcuts deliberately mirror Jira's own editor, so there is nothing new to learn.

Shortcuts below are written in Mac notation. On Windows and Linux read ⌘ as Ctrl, ⇧ as Shift and ⌥ as Alt. The app detects your machine and shows the right one in every tooltip.

Tables

The table button opens a size grid — drag across it to choose how many rows and columns you want. Once the cursor is inside a table, a second row of controls appears with everything you need to reshape it:

  • Header row and Header col — toggle either independently, so a table can have a header row, a header column, both, or neither.
  • Add columns before or after the current one, and add rows above or below.
  • Delete table to remove the whole thing in one click.

Code blocks

A code block keeps your indentation and monospacing intact, and carries a language selector with 42 languages — Bash, C, C++, C#, CSS, Dart, Go, GraphQL, Groovy, Haskell, HTML, Java, JavaScript, JSON, Kotlin, Python, Ruby, Rust, SQL, TypeScript, YAML and more, plus (None) for plain text. The language travels with the comment, so Jira highlights it correctly on every ticket you post to.

Mentions, links and emoji

  • @mentions — type @ and search your Jira users. Mentioned people are notified on every ticket the comment lands on, which is why a send containing a mention always asks you to confirm first.
  • Smart links — paste a Jira or Confluence URL and it converts into a live link card showing the item's title, status, priority and assignee, fetched fresh rather than frozen at paste time.
  • Emoji — browse from the toolbar or type /emoji.

Attachments

Attach files three ways: drag them into the editor, paste a screenshot straight from the clipboard, or use the toolbar button. Dropping a file places it where your pointer is, not where the cursor happened to be.

  • Up to 150 MB per file, with no limit on how many files one comment can carry. If your Jira site sets a lower attachment limit, that one wins — the app reads the site's real limit and checks against it the moment you attach, so an oversized file is refused immediately rather than after you have written the comment.
  • Uploading starts on attach, not on Send. The file moves while you are still typing, so the send itself is usually near-instant. Nothing is visible on any ticket until you actually send, and anything you attach and then delete is discarded.
  • Images and video preview inline, align left, centre or right, and resize by dragging the edge — the same handles Jira gives you. PDFs and other documents post as file cards.
  • Large files are copied per ticket in the background, so a long video posting to 25 tickets never stalls or times out the batch.

Copying a comment

Instead of writing something new, this copies a comment that already exists — including any images, videos and file attachments on it — onto other tickets exactly as-is.

You need two things: the tickets that should receive it, and a link to the comment being copied. The order is up to you. Grab the link from the source ticket first and then open the app, or open the app, pick your tickets, and fetch the link from another tab when you get to it.

Either way it lands in the same place: select the tickets in the Tickets step, then on the Comment step switch from Write a comment to Copy a comment and paste the link there. Send behaves exactly as it does for a comment you wrote.

You don’t have to memorise any of this. Switching to Copy a comment puts the whole procedure on screen, worked example included — and the Copy to n tickets button stays disabled until what you paste is actually a comment link:

The only part that happens outside the app is step 1. The link icon appears when you hover a comment in Jira, tucked beside the author’s name — it is easy to miss:

A plain issue link will not work. …/browse/KAN-6 points at the ticket, not at a comment on it. The link has to carry ?focusedCommentId= — that parameter is the only thing identifying which comment to copy.

Choosing Tickets

In the Tickets step, search by keyword or issue key, filter by project, or use advanced JQL to build a precise target list. Select up to 25 tickets. This is always the first step, whether you go on to write a comment or copy one.

Sending & Delivery

The Send step tracks every ticket individually and posts them in parallel, so a batch of 25 goes out together rather than one after another. Three counters — Delivered, Sending, Failed — update live above a row for every ticket, and each row links straight to the comment on its issue. When the run finishes the heading changes to Comments posted and tells you how long it took.

Anything that fails gets a Retry n failed button that re-runs only those tickets, leaving the delivered ones untouched.

Undo is there if you need it. Undo — delete comments and files removes the comments that send posted and the files it uploaded, freeing that storage again. Files that were already on a ticket beforehand are left alone.

Heads-up before a big send: posting to 10 or more tickets — or any comment that @mentions someone — shows a quick confirmation first, since every ticket notifies its watchers and mentioned users.

Jumping In From an Issue

You don't need to start from the global page. Two in-context surfaces let you act without leaving the ticket you're on:

  • ••• actions menu → Bulk Comments — opens the comment editor already seeded with the current ticket, so you can write a comment and add more tickets to post it to.
  • Activity panel → Bulk Comments tab — auto-selects that ticket's most recent comment so you can copy it, media and all, onto other tickets.

  • Modern web browser (Chrome, Firefox, Safari, Edge)
  • Active internet connection
  • Appropriate Atlassian Marketplace license

Permissions Required

Four scopes, and no others:

  • read:jira-work allowImpersonation — read the issues and comments you select, so they can be listed, previewed and copied.
  • write:jira-work allowImpersonation — post the comments and upload their attachments.
  • read:jira-user — resolve names for @mentions and show who authored a comment.
  • storage:app — the app's own storage on Atlassian's platform, used to hold per-ticket job status while a send runs and to stage attachments on their way to Jira. No issue content is kept there.

The two Jira scopes are granted with impersonation, which is what lets the app act as you rather than as itself. It is the reason comments and files show your name as the author on every ticket, including the large attachments that finish uploading in the background after you have moved on.

Cloud only: Bulk Comments for Jira is built on Atlassian Forge and runs exclusively on Jira Cloud. Jira Server and Jira Data Center are not supported.

Installation

Installing takes about a minute and needs a Jira Cloud site admin, since apps are installed site-wide.

  1. Open Bulk Comments for Jira on the Atlassian Marketplace, or search for it from Apps → Explore more apps inside Jira.
  2. Click Try it free and choose your Jira Cloud site.
  3. Accept the permissions it asks for: read and write Jira work items, read Jira user profiles, and app storage. Nothing else is requested — see Permissions Required for what each one does.
  4. Open it from Apps in the left sidebar, or from any issue's ••• actions menu and Activity panel.

Free for teams of 10 or fewer. Above that, paid tiers are billed through the Atlassian Marketplace on your existing Atlassian invoice — there is no separate DataPingo account to create and no card to enter with us.

Removing the app is just as clean. Uninstalling from Apps → Manage apps deletes everything it stored, and comments already posted stay on their tickets as ordinary Jira comments.
