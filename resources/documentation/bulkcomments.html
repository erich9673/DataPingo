Bulk Comments for Jira

Overview

Bulk Comments for Jira is an Atlassian Forge app installed directly inside Jira Cloud. It lets you compose a rich-text comment once and post it to up to 25 issues at once, or replicate an existing comment — including its images, videos and file attachments — onto multiple tickets exactly as-is.

No context switching: Everything happens inside Jira. There's no external server in the data path for Compose or Replicate — comments and media move directly between Jira tickets through Atlassian's own infrastructure.

Compose

Write a new rich-text comment — bold, lists, @mentions — and post it to up to 25 Jira issues in one action.

Replicate

Paste the link to an existing comment and copy it exactly as-is onto other tickets. Images, videos and file attachments are copied automatically.

Three Entry Points

Open the app from the global page in the left sidebar, the "Bulk Comments for Jira" tab in any issue's Activity panel, or the "Bulk Comment" option in an issue's ••• actions menu. The latter two auto-select that ticket's most recent comment for a fast copy.

Full Guide

Installing the App

  1. Go to the Atlassian Marketplace and search for Bulk Comments for Jira.
  2. Click Get it now and select your Jira Cloud site.
  3. Accept the required permissions: read & write Jira work items, read Jira user profiles.
  4. Once installed, find Bulk Comments for Jira in the left-side Apps menu, or look for it on any issue's Activity panel and ••• actions menu.

Compose Mode

Write a new comment with rich-text formatting — bold, italics, lists, and @mentions — and choose the issues to post it to. The comment is posted individually to each selected ticket.

Replicate Mode

Instead of writing something new, Replicate mode copies a comment that already exists — including any images, videos and file attachments on it — onto other tickets exactly as-is.

Getting the comment link: Open the source comment's ••• menu on the Jira issue and choose Copy link, then paste it into Bulk Comments for Jira's Replicate field.

Selecting Tickets

Search by keyword or issue key, filter by project, or use advanced JQL to build a precise target list. Select up to 25 tickets, then send. Per-ticket progress is tracked live, and any failures can be retried in one click.

Jumping In From an Issue

You don't need to start from the global page. Open any issue's ••• actions menu and choose Bulk Comment, or switch to the Bulk Comments for Jira tab in the issue's Activity panel — both auto-select that ticket's most recent comment so you can jump straight into Replicate mode.

System Requirements

  • Active Jira Cloud instance (Jira Software or Jira Work Management)
  • Modern web browser (Chrome, Firefox, Safari, Edge)
  • Active internet connection
  • Appropriate Atlassian Marketplace license

Permissions Required

  • Read and write Jira work items (to fetch tickets and post comments)
  • Read Jira user profiles (for @mentions and issue metadata)

Cloud only: Bulk Comments for Jira is built on Atlassian Forge and runs exclusively on Jira Cloud. Jira Server and Jira Data Center are not supported.

Privacy & Security

  • Data Privacy: DataPingo Privacy Policy
  • Security: DataPingo Security Policy
  • Data Storage: All app data is stored in Atlassian Forge Storage — no data is sent to or stored on external DataPingo servers
  • Media Handling: Images, videos and files are copied directly between Jira tickets through Atlassian's infrastructure — never through an external server DataPingo operates
  • Compliance: Follows Atlassian security standards and the Forge platform security framework
