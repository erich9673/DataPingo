📋 Overview

Bulk Page Cloner is a powerful Atlassian Forge app designed for Confluence Cloud that enables users to create multiple pages simultaneously, streamlining documentation workflows and project setup processes.

💡 Pro Tip: This tool is perfect for teams that need to maintain consistency across multiple Confluence spaces or quickly replicate content structures for new projects.

📘 Full Guide

Open the App (3 ways)

  1. Click Bulk Page Cloner for Confluence under Apps on the left side of your screen.
  2. Type /bulkpages in any Confluence page and select Bulk Page Cloner for Confluence.
  3. Open the app under the ••• menu in a parent page and select Create Bulk Page(s).

Step 1 — Select a Page as a Template

Pick exactly which Confluence page you will use to create multiple copies.

  • Use the search bar to find any Confluence page within your environment and select it.
  • If you can’t find it, use the 🔗 Paste URL option and paste the desired Confluence link.
  • You can also use the Space filter to locate your page.
  • This will load up to 5,000 Confluence Pages

Step 2 — Bulk Cloning

Choose how many pages to create and what to name them.

  1. Use the red slider and choose from 1–25 pages.
  2. Write the names of the new page(s).

Features

  • 📋 Copy First Title to All — Quickly duplicate the first title across all fields.
  • 🗑️ Clear All — Deletes all title fields.
  • Autofill — If you name a page “Report1”, it will prompt to auto-populate “Report2”, etc. Works for weekly/monthly date patterns too.

Step 3 — 📍 Location

Choose where to place your pages.

  1. 📁 Select Confluence Space — Select a Space.
  2. Page Creation Options:
    • Create Child Page — Add reports as child pages under an existing parent page.
    • Create Parent Page — Create a new parent page for organizing your reports.
    • Create Parent & Child — Create both a new parent page and child pages structure.

Step 4 — Success Page

You will see the generated pages with links to each one.

🧰 Technical Specifications

System Requirements

  • Confluence Cloud instance
  • Modern web browser (Chrome, Firefox, Safari, Edge)
  • Active internet connection
  • Appropriate Confluence permissions

Compatibility

  • Atlassian Product: Confluence Cloud
  • Supported Versions: All current Confluence Cloud versions
  • Forward Compatibility: Automatically compatible with future Cloud updates

Performance

  • Optimized for bulk operations
  • Handles up to 5000 pages per batch via browser interface
  • URL-based fallback for larger operations
  • Minimal impact on Confluence performance

Permissions Required

  • Space administration rights (for space-level operations)
  • Page creation permissions
  • Template access rights
  • Read/write access to target spaces

Troubleshooting

Pages Not Creating

  • Verify you have page creation permissions in the target space
  • Check that the selected template is available
  • Ensure page titles don't contain invalid characters

Template Not Loading

  • Confirm the template exists and is accessible
  • Check space-level template permissions
  • Refresh the app interface and try again

Slow Performance

  • Reduce batch size for better performance
  • Check network connection stability
  • Consider creating pages in smaller batches

Access Denied Errors

  • Verify Confluence space permissions
  • Contact your Confluence administrator for access rights
  • Ensure the app has proper permissions in your instance

Best Practices

Planning Your Page Structure

  • Plan your page hierarchy before bulk creation
  • Use consistent naming conventions
  • Consider future organization needs

Template Selection

  • Choose appropriate templates for your content type
  • Test with a small batch before large operations
  • Ensure templates meet your formatting requirements

Performance Optimization

  • Create pages in batches of 50-100 for optimal performance
  • Avoid special characters in page titles
  • Use numbered sequences for systematic organization
